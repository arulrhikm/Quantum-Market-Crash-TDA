Create Your IBM Quantum Account

~3 minutes
Why this matters
IBM Quantum gives you free access to real quantum hardware — actual superconducting qubits cooled to 15 millikelvin inside a dilution refrigerator. Your circuit will execute on that hardware, not a simulator. This is what makes the title "Quantum Computation" genuinely true.
  • Open your browser and go to IBM Quantum

    Navigate to the URL below. You need to get to the account creation page.

🔗 quantum.cloud.ibm.com
  • Click "Create an IBMid" or "Sign Up"

    It's free. You do not need a credit card. Use your personal or school email — it doesn't matter which.

  • Fill in your name, email, and create a password

    Password must be at least 8 characters. Write it down — you'll need it again.

  • Check your email inbox for a verification link from IBM

    Subject line: "Verify your IBMid". Click the link inside. If you don't see it in 2 minutes, check spam.

  • Log in and complete any onboarding prompts

    IBM may ask what you're using it for — just say "research / education". Accept terms of service.

✅
Checkpoint: You should be looking at the IBM Quantum dashboard

You'll see a page with "Compute resources", maybe some news, and your account name in the top-right corner. If you're here, Step 1 is done.

2

Copy Your API Token

~2 minutes
⚠ Important
Your API token is like a password. Do not paste it into any public website, GitHub repo, or share it with anyone. You'll only paste it into the Python file on your own computer.
  • Click your profile icon in the top-right corner of the dashboard

    It looks like a person silhouette or shows your initials. A dropdown menu will appear.

  • Click "Account settings" or "Manage account"

    This takes you to your account management page at quantum.ibm.com/account

  • Scroll down to the section labeled "API token" or "Access token"

    It's usually about halfway down the page. You'll see a long string of letters and numbers, partially hidden.

  • Click "Copy" or the copy icon next to the token

    The token is a long string that looks like: a8f3c2d1e9b4...7f0e2a1b3c4d (much longer in reality — about 200 characters). If you only see asterisks, click "Reveal" first, then copy.

  • Paste it somewhere safe temporarily

    Open Notepad (Windows), TextEdit (Mac), or any plain text editor. Paste it there so you don't lose it. You'll move it to the Python file in Step 4.

🔑
Your token will look like this
a8f3c2d1e9b47f0e2a1b3c4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2b3c4d5e6...
✅
Checkpoint: You have the token copied

It should be a very long alphanumeric string, at least 100 characters. If it's shorter than that, you may have copied something else — go back and look again.

3

Install Required Libraries

~5 minutes
You need two Python packages: qiskit (builds the circuit) and qiskit-ibm-runtime (talks to IBM's servers). You already have numpy/scipy from the main project.
  • Open a Terminal (Mac/Linux) or Command Prompt (Windows)

    Mac: Press Cmd+Space, type "Terminal", press Enter.   Windows: Press Win+R, type "cmd", press Enter.

  • Run this exact command and press Enter

    This installs both packages. It may take 1–3 minutes.

pip install qiskit qiskit-ibm-runtime matplotlib
  • Wait for it to finish — you'll see "Successfully installed" at the end

    It downloads several packages. You'll see a lot of text scrolling by. That's normal. Look for the green "Successfully installed" message at the very end.

  • Verify the installation worked

    Run this quick check. If you see version numbers printed, you're good. If you see an error, try pip3 instead of pip.

python -c "import qiskit; import qiskit_ibm_runtime; print('Qiskit:', qiskit.__version__)"
Expected output
Qiskit: 1.x.x
Runtime: 0.x.x
If you see "pip is not recognized"
You need to install Python first. Go to python.org/downloads, download Python 3.11 or later, install it, then come back and run the pip command again. Make sure to check "Add Python to PATH" during installation.
✅
Checkpoint: Both packages installed and verified

You see version numbers when running the verify command. No error messages.

Configure the Hardware Validation File

~3 minutes
You need to open hardware_validation.py (the file from this project) and paste your IBM token into it. That's the only change you need to make.
  • Find hardware_validation.py on your computer

    It's in the MSEF_2026 folder you downloaded from this project. If you're not sure where it is, search your Downloads folder.

  • Open it in any text editor

    Right-click the file → "Open with" → choose Notepad (Windows), TextEdit (Mac), or VS Code if you have it. Do NOT open it in Word.

  • Find line 174 — it looks exactly like this

    Use Ctrl+F (Find) and search for YOUR_IBM_TOKEN_HERE to jump right to it.

# PASTE YOUR IBM TOKEN HERE:
IBM_TOKEN = "YOUR_IBM_TOKEN_HERE"
  • Replace YOUR_IBM_TOKEN_HERE with your actual token

    Keep the quotation marks. It should look like this after:

IBM_TOKEN = "a8f3c2d1e9b47f0e2a1b3c4d5e6f7a8b..." # your real token here
  • Save the file

    Press Ctrl+S (Windows) or Cmd+S (Mac). Make sure you save it as .py — not .py.txt. If using Notepad, choose "Save as" → set file type to "All files" → name it hardware_validation.py.

✅
Checkpoint: Token is in the file, file is saved

Open the file again and verify the token is there between the quotation marks on the IBM_TOKEN line. The token should be very long — at least 100 characters.

Run the Circuit

~10–20 minutes (most of this is queue wait time)
What this does
The script will: (1) build the QPE circuit, (2) connect to IBM's servers, (3) find the least-busy real quantum device, (4) submit your job to the queue, (5) wait for results, (6) compare hardware vs simulation, (7) save a figure and JSON file.
  • Open Terminal / Command Prompt

    Same as Step 3.

  • Navigate to the folder containing hardware_validation.py

    Use the cd command. For example, if the file is in your Downloads/MSEF_2026 folder:

# Windows example:
cd C:\Users\YourName\Downloads\MSEF_2026

# Mac example:
cd ~/Downloads/MSEF_2026
  • Run the script

    Type this and press Enter. Keep the terminal window open — do not close it.

python hardware_validation.py

Collect Your Output Files

~2 minutes
The script generates three output files in the same folder as hardware_validation.py. You need all three for your project.
  • hardware_results.json — your raw data

    Contains: backend name, job ID, shot count, raw measurement counts, comparison statistics (TVD, Hellinger distance), β₀ estimate from hardware. This is your evidence of real hardware execution.

  • figure5_hardware.png — your comparison plot

    Side-by-side bar chart of simulation vs hardware measurement distributions, plus a deviation panel. Use this directly in your poster and paper.

  • Your terminal output — screenshot this

    Take a screenshot of the terminal showing the comparison table, TVD value, and β₀ estimate. This is additional evidence.

Interpret Your Results for Judges

Read carefully — this is what you'll say at the fair

Your hardware result will fall into one of three categories. Here's what each means and exactly how to explain it to a judge:

✓ Strong Agreement

TVD < 0.15

Hardware and simulation closely match. β₀ estimate ≈ 1. Means the IBM device had low noise on the day you ran it. Tell judges: "The hardware reproduced the simulation within X% — strong validation of the algorithm."

~ Moderate Noise

TVD 0.15–0.35

Some deviation due to NISQ gate errors and decoherence. This is the most common case. Tell judges: "TVD of X confirms NISQ noise is present — exactly what the limitations section predicts. Fault-tolerant hardware would reduce this toward 0."

✗ High Noise

TVD > 0.35

Significant decoherence. β₀ estimate may be far from 1. This is also a valid result — it quantitatively demonstrates why fault tolerance matters. Tell judges: "The high TVD directly validates Limitation #3 in my paper."

! What if it errors out?

Connection / queue errors

Try again at a different time. If IBM's backend is down, wait a few hours. If you get a token authentication error, double-check you copied the full token with no extra spaces.

Exactly what to say when a judge asks "did this run on a quantum computer?"

✓ Your Answer
"Yes. I submitted a 4-qubit QPE circuit to IBM's [backend name] quantum processor, Job ID [your job ID], which you can verify at quantum.cloud.ibm.com/jobs. The device uses superconducting transmon qubits at 15 millikelvin. The hardware returned 1,024 measurements with a Total Variation Distance of [your TVD] from the ideal simulation — quantifying the NISQ noise. My β₀ estimate from hardware was [your value] versus the classical truth of 1. This gap directly validates the limitations section of my paper."
